# AI Agent Instructions

This repository may be modified by AI coding tools.

Follow these rules strictly.

## Do Not Overengineer

Avoid unnecessary complexity.

Do not introduce frameworks unless explicitly required.

## Backend Rules

Backend must use:

- Go
- Go standard library
- `net/http`
- SQLite

Do not add:

- Gin
- Echo
- Fiber
- ORM libraries

## Frontend Rules

Frontend must remain simple.

Allowed:

- HTML
- CSS
- minimal JavaScript

Do not add:

- React
- Vue
- Angular
- Next.js
- Node.js
- npm

## Architecture Rules

Follow the project structure defined in `docs/ARCHITECTURE.md`.

Do not change folder layout without good reason.

## Code Quality

Code must be:

- readable
- minimal
- well structured

Avoid large abstractions.

Prefer simple functions.

## Data Handling Rules

Quran text must be loaded from `data/quran.json`.

The database stores only verse relations.

For Quran dataset updates:

- do not alter Quran text content
- preserve UTF-8 Arabic text and diacritics
- keep attribution and source notes in `NOTICE.md`
- validate dataset integrity before committing

Required dataset checks:

- exactly `6236` ayah records
- unique `(surah, ayah)` pairs
- non-empty `text_ar`

## UI Rules

Always display full verses.

Never truncate Quran text.

Arabic text must be readable.

## Development Strategy

Implement features incrementally:

1. HTTP server
2. load Quran dataset
3. verse search
4. relations API
5. frontend pages
6. compare view

## Open Source Hygiene

When making user-visible changes:

- update relevant docs (`README.md`, `docs/PROJECT.md`, `docs/ARCHITECTURE.md`)
- follow `CONTRIBUTING.md`
- record user-visible changes in `CHANGELOG.md`

## Commit Attribution Policy

This project accepts code from many tools and models. Git history must stay neutral and human-owned.

Required:

- write concise, human-style commit messages focused on the actual change
- keep commit metadata model-agnostic
- remove accidental AI marketing or co-author trailers before pushing

Do not include commit-message trailers or lines such as:

- `Co-authored-by: Claude ...`
- `Generated with Claude Code`
- `Co-authored-by: ChatGPT ...`
- `Generated by <AI tool/model> ...`

If any tool auto-adds those lines, disable that behavior in tool settings and amend the commit message before push.

If already committed, clean it up with history rewrite:

- latest commit only:
  - `git commit --amend`
  - remove attribution lines in editor, save, then:
  - `git push --force-with-lease`
- older commits:
  - `git rebase -i <base-commit>`
  - mark target commits as `reword` (or `edit`), remove attribution lines, then:
  - `git rebase --continue`
  - `git push --force-with-lease`

Safety rules:

- announce history rewrite to collaborators before force-push
- prefer `--force-with-lease` (never plain `--force`)
- if attribution appears in many commits, use `git filter-repo` in a dedicated maintenance branch and review carefully before replacing shared history

## Priority

The primary goal is usability for Quran memorization.

Keep the interface simple and fast.
